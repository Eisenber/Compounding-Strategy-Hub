# README_AGENT

## 任务

基于 `agent-mvp-plan.md`，生成一个**可运行的最小 MVP**：

- 前端：一个可用的复利横向对比网页
- 后端：一个可用的 Java API 服务
- 联调后：用户可以输入多个方案并看到对比结果

这是一个**A 股小白投资者**使用的工具型产品，不是量化平台。

---

## 核心目标

你唯一需要完成的目标是：

**让用户完成一次真实可用的复利方案横向对比。**

如果某个设计、代码或依赖不能直接帮助达成这个目标，就不要做。

---

## 必须遵守的范围

只做这些：

- 首页
- 复利对比页
- Java 后端健康检查接口
- Java 后端复利对比接口
- 前后端联调
- 本地启动说明

不要做这些：

- 选股
- 实时行情
- 回测
- 登录注册
- 用户中心
- 数据库优先设计
- 管理后台
- 推荐买卖
- AI 投顾文案

---

## 实施原则

1. 先跑通，再优化
2. 先单接口闭环，再补样式
3. 先写最少文件，再考虑抽象
4. 输入输出结构必须稳定
5. 所有计算口径以后端为准

---

## 推荐实现方式

### 后端

- Java 21
- Spring Boot
- Maven
- REST Controller

建议做法：

- 不强制接数据库
- 计算逻辑放在 service
- DTO 和返回对象单独定义
- 接口返回统一 JSON

### 前端

- React 方案即可
- 使用简单图表库
- 表单可直接本地 state 管理

建议优先：

- Vite + React
- Tailwind CSS
- ECharts 或 Recharts

如果 agent 判断 `Next.js` 更顺手，也可以，但不要因为框架复杂度拖慢 MVP。

---

## 最小交付物

必须产出这些内容：

1. `backend/`
   - 可启动的 Spring Boot 项目
   - 至少 2 个接口

2. `frontend/`
   - 可启动的网页项目
   - 至少 2 个页面或 1 个首页 + 1 个主页面

3. `README.md`
   - 明确说明前后端如何启动

4. 可联调结果
   - 前端点击按钮后能展示后端返回的结果

---

## 后端最低要求

必须实现：

### `GET /api/health`

返回：

```json
{ "status": "ok" }
```

### `POST /api/v1/compound/compare`

必须支持：

- 2 到 5 个方案输入
- 参数校验
- 返回每个方案的结果
- 返回年度曲线数据
- 返回整体结论

后端必须完成的计算字段：

- `totalInvested`
- `finalAmount`
- `totalProfit`
- `profitMultiple`
- `realFinalAmount`
- `yearlyPoints`

---

## 前端最低要求

页面必须包含：

### 首页

- 标题
- 简介
- 进入复利对比页按钮

### 复利对比页

- 默认显示 2 个方案
- 允许新增方案，最多 5 个
- 允许删除方案
- 每个方案有完整输入字段
- 点击“开始对比”后调用后端
- 显示结果卡片
- 显示折线图
- 显示 2 到 3 条结论

## 执行顺序

严格按这个顺序：

1. 创建后端项目
2. 写健康检查接口
3. 写复利计算核心 service
4. 写 compare 接口
5. 本地验证后端接口
6. 创建前端项目
7. 搭输入表单
8. 接 compare 接口
9. 显示结果卡片
10. 显示曲线图
11. 补充错误提示和空状态
12. 写 README

不要一开始就做：

- 全局状态管理
- 国际化
- 复杂组件抽象
- 数据持久化

## 复利计算口径

必须统一使用以下规则：

- 月净收益率 = `(1 + annualReturnRate - annualFeeRate)^(1/12) - 1`
- 默认每月末定投
- 总投入 = 初始本金 + 每月定投金额 * 年限 * 12
- 实际购买力 = `finalAmount / (1 + inflationRate)^years`
- 收益倍数 = `finalAmount / totalInvested`

如果需要图表数据，至少按年输出资产节点。

## 验收标准

只有以下全部满足，才算完成：

- 前端能启动
- 后端能启动
- 健康检查接口可访问
- compare 接口可访问
- 页面可以输入多个方案
- 页面点击后能展示真实结果
- 图表正常渲染
- 非法输入有明确提示
- README 可以指导别人启动项目

## 自检清单

在结束前，agent 必须自己检查：

- 是否真的控制在 MVP 范围内
- 是否引入了不必要的依赖
- 是否存在只写了前端假数据、没有接真实接口的情况
- 是否存在接口字段和前端展示不一致的情况
- 是否存在计算逻辑写在前端、后端只透传的情况
- 是否写清楚了启动命令

---

## 输出风格要求

页面文案应保持：

- 简单
- 新手可理解
- 不夸大收益
- 不构成投资建议

避免出现：

- “稳赚”
- “高收益低风险”
- “必涨”
- “推荐买入”

## 完成定义

当用户可以在本地打开网页，输入例如：

- 银行定存
- 债基
- 沪深300 ETF

然后点击按钮，看到每种方案 10 年后的：

- 总投入
- 最终资产
- 总收益
- 实际购买力
- 曲线图

这件事完成了，MVP 就完成了。
