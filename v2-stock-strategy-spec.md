# 第二版规格说明与需求分析报告

## 1. 文档目的

本报告用于将“第二版接入选股策略”的模糊想法，扩展为可执行的产品规格说明，服务于后续的产品设计、研发排期、接口设计和验收。

本版本面向第一版 `复利对比工具` 稳定后的第二阶段建设，目标是在不进入复杂量化和回测系统的前提下，新增一个对 **A 股小白投资者** 友好的 `新手选股策略页`。

---

## 2. 背景与问题定义

第一版解决的是“长期投入方式横向对比”的问题，帮助用户理解：

- 每月投入金额的长期影响
- 年化收益率差异的长期影响
- 手续费与通胀对资产结果的侵蚀

但第一版仍然缺少一个自然延伸的问题：

**如果用户理解了长期投资和复利，下一步他会问：我可以看哪些 A 股标的？**

因此第二版的职责不是做复杂量化平台，而是提供一个：

- 简单
- 可解释
- 可筛选
- 有风险提示

的新手策略筛选工具。

---

## 3. 产品目标

## 3.1 核心目标

让用户基于几个简单、可解释的策略模板，在 A 股股票池中筛选出候选股票列表，并理解这些股票“为什么入选”。

## 3.2 非目标

第二版明确 **不做**：

- 自动交易
- 复杂多因子量化
- 高频策略
- 完整回测系统
- 实时盘口决策
- 个性化投顾
- 收益承诺或买卖推荐

## 3.3 成功标准

用户在 3 分钟内可以完成以下事情：

1. 选择一个策略模板
2. 查看默认筛选条件
3. 调整 1 到 3 个参数
4. 获取候选股票列表
5. 理解每只股票的核心入选原因和主要风险提示

---

## 4. 用户画像

目标用户：

- A 股新手投资者
- 对股票有初步兴趣
- 能理解 PE、PB、ROE、股息率等基础概念，但不擅长复杂财务分析

用户需求：

- 想快速找到“看起来值得进一步研究”的股票
- 不想一上来面对过多专业参数
- 希望页面给出“为什么入选”的解释
- 希望有明显的风险提醒，避免误以为系统在直接荐股

---

## 5. 产品定位

第二版的 `选股策略页` 定位为：

**A 股股票初筛工具**

它的作用是帮助用户完成“候选池筛选”，而不是替用户做最终决策。

页面最终输出的是：

- 候选列表
- 核心指标
- 入选原因
- 风险提示

而不是：

- 买点
- 卖点
- 仓位建议
- 收益预测

---

## 6. 功能范围

## 6.1 本期范围

第二版只做 1 个主页面：

- `选股策略页`

包含 4 个核心能力：

1. 选择策略模板
2. 查看和调整筛选条件
3. 获取股票候选列表
4. 查看指标解释、入选原因与风险提示

## 6.2 页面输出内容

每只股票至少展示：

- 股票代码
- 股票名称
- 所属行业
- 当前价格或最近收盘价
- 市盈率 PE
- 市净率 PB
- ROE
- 营收增长率
- 净利润增长率
- 股息率
- 市值
- 入选原因
- 风险标签

## 6.3 暂不纳入范围

- 自选股跟踪
- 用户收藏策略
- 导出 Excel 以外的高级报表
- 组合管理
- 详细回测页
- 多市场支持

## 7. 策略模板设计

为保证小白可理解，首期只提供 3 个模板。

## 7.1 低估值策略

适用人群：

- 更关注“价格相对便宜”的用户

核心逻辑：

- PE 较低
- PB 较低
- 排除明显基本面恶化公司

建议默认规则：

- `PE(TTM) <= 15`
- `PB <= 1.8`
- `净利润增长率 >= 0`
- `ROE >= 8%`
- 排除 `ST`
- 排除停牌
- 排除上市未满 180 天股票

页面解释文案：

- 该策略优先筛选估值相对较低、盈利未明显恶化的公司

主要风险：

- 低估值不等于一定会上涨
- 可能存在周期行业、业绩下滑或价值陷阱

## 7.2 高股息策略

适用人群：

- 偏好稳健分红、希望关注现金流回报的用户

核心逻辑：

- 股息率较高
- 分红相对稳定
- 盈利能力不能过弱

建议默认规则：

- `股息率 >= 4%`
- `近 3 年至少有连续分红记录`
- `ROE >= 8%`
- `资产负债率 <= 70%`
- 排除 `ST`
- 排除停牌

页面解释文案：

- 该策略优先筛选股息率较高且分红相对稳定的公司

主要风险：

- 高股息可能来自股价下跌
- 分红稳定不代表未来分红不变

## 7.3 质量成长策略

适用人群：

- 偏好业绩质量和成长性的用户

核心逻辑：

- ROE 较高
- 营收和利润保持增长
- 负债结构不过分激进

建议默认规则：

- `ROE >= 12%`
- `营收增长率 >= 15%`
- `净利润增长率 >= 15%`
- `资产负债率 <= 60%`
- `PE(TTM) <= 40`
- 排除 `ST`
- 排除停牌

页面解释文案：

- 该策略优先筛选盈利质量和成长性表现较好的公司

主要风险：

- 高成长通常伴随更高估值和波动
- 成长延续性存在不确定性

---

## 8. 筛选规则与解释机制

## 8.1 筛选逻辑要求

每个策略都必须满足：

- 有默认阈值
- 支持用户微调
- 支持按全部条件交集筛选
- 支持排序

## 8.2 入选原因生成

每只股票返回的“为什么入选”必须可解释，不能只返回冷冰冰的指标。

建议生成规则：

- 命中低估值策略：
  - “PE 和 PB 低于默认阈值，同时 ROE 达到基本要求”
- 命中高股息策略：
  - “股息率高于阈值，且近年保持分红记录”
- 命中质量成长策略：
  - “ROE、营收增长和净利润增长同时达到成长条件”

## 8.3 风险标签生成

建议按规则自动打标签，例如：

- `高估值风险`
- `高负债风险`
- `周期行业风险`
- `分红波动风险`
- `增长放缓风险`

---

## 9. 页面与交互说明

## 9.1 页面结构

建议页面分为 5 个区块：

1. 顶部说明区
   - 页面标题
   - 一段说明：本工具仅用于股票初筛，不构成投资建议

2. 策略模板区
   - 低估值
   - 高股息
   - 质量成长

3. 筛选条件区
   - 展示当前模板默认条件
   - 允许用户微调阈值
   - 支持恢复默认参数

4. 结果列表区
   - 表格展示候选股票
   - 支持排序
   - 支持分页

5. 说明与风险区
   - 指标解释
   - 风险提示
   - 使用说明

## 9.2 关键交互

- 点击策略模板后，自动加载该模板默认条件
- 修改条件后点击“开始筛选”
- 结果返回后显示候选数
- 点击表头支持排序
- 点击某只股票可展开更多解释

---

## 10. 数据需求分析

## 10.1 所需基础字段

第二版至少需要以下数据字段：

- 股票代码
- 股票名称
- 行业
- 是否 ST
- 是否停牌
- 上市日期
- 最新价或最近收盘价
- 市盈率 PE(TTM)
- 市净率 PB
- ROE
- 营收增长率
- 净利润增长率
- 股息率
- 市值
- 资产负债率
- 分红记录

## 10.2 数据口径要求

需要统一口径，避免页面解释和筛选结果不一致：

- PE 优先使用 `TTM`
- ROE 使用最近年度或最近可用财报口径
- 增长率优先使用同比
- 股息率需要明确是静态还是近 12 个月口径

建议首版统一为：

- `PE(TTM)`
- `PB(LF)` 或最近一期
- `ROE(最近年度)`
- `营收同比增长`
- `净利润同比增长`
- `近 12 个月股息率`

## 10.3 数据更新频率

建议：

- 行情类字段：日更
- 财务类字段：随财报更新
- 分红类字段：随公告或日更汇总

---

## 11. 数据源建议

考虑后端使用 Java，建议优先采用适合 HTTP 接入的数据源。

优先方案：

- TuShare API

备选方案：

- 自建 Python 数据采集服务后供 Java 调用
- AKTools 作为 HTTP 服务层

数据源选择标准：

- 字段覆盖完整
- 能稳定拿到 A 股基础面指标
- 能支持每日更新
- 对 Java 接入友好

---

## 13. 接口规格建议

## 13.1 获取策略模板列表

`GET /api/v2/strategies/templates`

返回：

```json
[
  {
    "code": "LOW_VALUATION",
    "name": "低估值",
    "description": "筛选估值较低且盈利未明显恶化的股票"
  },
  {
    "code": "HIGH_DIVIDEND",
    "name": "高股息",
    "description": "筛选股息率较高且分红相对稳定的股票"
  },
  {
    "code": "QUALITY_GROWTH",
    "name": "质量成长",
    "description": "筛选盈利质量和成长性较好的股票"
  }
]
```

## 13.2 获取某个模板默认条件

`GET /api/v2/strategies/templates/{code}`

返回示例：

```json
{
  "code": "LOW_VALUATION",
  "name": "低估值",
  "filters": {
    "maxPe": 15,
    "maxPb": 1.8,
    "minRoe": 8,
    "minProfitGrowth": 0,
    "excludeSt": true,
    "excludeSuspended": true,
    "minListingDays": 180
  }
}
```

## 13.3 执行策略筛选

`POST /api/v2/strategies/screen`

请求示例：

```json
{
  "strategyCode": "HIGH_DIVIDEND",
  "filters": {
    "minDividendYield": 4,
    "minRoe": 8,
    "maxDebtRatio": 70,
    "excludeSt": true,
    "excludeSuspended": true
  },
  "sortBy": "dividendYield",
  "sortDirection": "desc",
  "page": 1,
  "pageSize": 20
}
```

返回示例：

```json
{
  "total": 86,
  "page": 1,
  "pageSize": 20,
  "items": [
    {
      "symbol": "600xxx",
      "name": "示例股份",
      "industry": "银行",
      "price": 8.56,
      "peTtm": 6.8,
      "pb": 0.72,
      "roe": 10.6,
      "revenueGrowth": 4.2,
      "profitGrowth": 5.1,
      "dividendYield": 5.4,
      "marketCap": 1200.5,
      "reason": "股息率高于阈值，且近年保持分红记录",
      "riskTags": ["周期行业风险"]
    }
  ]
}
```

## 14. 结果排序需求

支持以下排序字段：

- 股息率
- PE
- PB
- ROE
- 营收增长率
- 净利润增长率
- 市值

默认排序建议：

- 低估值：`PE 升序`
- 高股息：`股息率降序`
- 质量成长：`ROE 降序` 或 `利润增长率降序`

---

## 15. 校验与约束

## 15.1 输入校验

需要限制：

- 页码合法
- 页大小上限，例如 `20` 或 `50`
- 指标阈值必须在合理范围内
- 最小值不能大于最大值

## 15.2 业务约束

- 模板编码必须有效
- 如果结果为空，返回空列表和友好提示
- 不允许页面显示“推荐买入”类措辞

---

## 16. 风险与合规要求

该页面存在较强金融误导风险，因此必须显式加入以下约束：

- 页面顶部长期展示：`本工具仅用于股票筛选与信息展示，不构成投资建议`
- 不承诺收益
- 不输出买卖指令
- 不生成“必涨”“低风险高收益”等表述

需要在结果页中提醒：

- 指标筛选只能帮助缩小研究范围
- 财务指标有滞后性
- 单一策略可能在某些市场环境下失效

---

## 17. 非功能需求

## 17.1 性能

- 单次筛选接口响应时间目标：`2 秒内`
- 在候选池不大的情况下，首版可接受 `3 秒内`

## 17.2 可用性

- 页面应支持新手阅读
- 指标名称旁建议有解释提示
- 空结果要给出可操作建议，例如“适当放宽条件”

## 17.3 可维护性

- 模板规则要可配置
- 筛选逻辑不要写死在前端
- 文案解释尽量后端统一生成，减少前后端不一致

---

## 18. MVP 与后续演进边界

## 18.1 第二版 MVP

只做：

- 3 个策略模板
- 默认条件可微调
- 候选股票列表
- 入选原因
- 风险标签

## 18.2 第三版可考虑

- 策略收藏
- 自选股观察
- 指数/行业筛选
- 基础历史表现查看
- 简版回测

## 18.3 暂不建议过早投入

- 多因子打分引擎
- 复杂回测框架
- 自动调仓模拟
- 智能买卖建议

---

## 19. 验收标准

满足以下条件时，第二版可判定为完成：

- 用户能看到 3 个策略模板
- 用户能查看和修改模板默认条件
- 用户能发起筛选
- 系统返回候选股票列表
- 每只股票都有核心指标展示
- 每只股票都有明确“为什么入选”
- 页面有显式风险提示
- 不出现荐股式表述

---

## 20. 研发建议与排期顺序

建议按以下顺序推进：

1. 明确数据字段与数据源
2. 定义策略模板结构
3. 完成筛选接口
4. 完成入选原因与风险标签规则
5. 开发前端策略页
6. 接入排序与分页
7. 增加指标解释和风险提示
8. 做联调与验收

---

## 21. 结论

第二版最合适的落点不是“高级量化”，而是一个对 A 股新手友好的 `股票初筛工具`。

该工具的产品价值在于：

- 帮用户快速缩小研究范围
- 用简单规则替代混乱的信息搜索
- 用解释与风险提示降低误用风险

因此，第二版的设计原则应始终保持：

**简单、可解释、可筛选、不荐股。**
