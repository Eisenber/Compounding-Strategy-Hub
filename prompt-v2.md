# Agent Prompt V2

你现在要在当前项目中开发“第二版功能”，但**不得破坏第一版**。

## 核心任务

在现有项目基础上，新增一个面向 **A 股小白投资者** 的 `第二版选股策略功能`，包含：

- 前端页面
- 后端接口
- 前后端联调

目标不是重做项目，而是：

**在保留第一版复利对比工具完整可用的前提下，新增第二版能力。**

---

## 第一优先级约束

必须严格遵守：

1. 不删除现有文件
2. 不重写第一版已有功能
3. 不改变第一版已有接口行为
4. 不改变第一版页面行为
5. 不以“重构”为名修改第一版核心逻辑
6. 第二版必须通过新增代码实现
7. 如果需要改老文件，只允许做“接入第二版入口”的最小改动

如果你认为某处必须修改旧代码，必须满足：

- 改动范围最小
- 不影响第一版输出
- 改动理由明确

---

## 第二版范围

第二版只做一个主能力：

`新手选股策略页`

页面能力包括：

- 策略模板选择
- 条件筛选
- 股票结果列表
- 指标解释
- 入选原因
- 风险提示

---

## 第二版产品目标

让用户完成这件事：

1. 选择一个策略模板
2. 查看默认筛选条件
3. 调整少量参数
4. 获取候选股票列表
5. 理解为什么这些股票入选
6. 看到风险提示

第二版不是：

- 自动交易系统
- 回测平台
- 智能投顾
- 实时行情终端

---

## 策略模板

第二版先只做 3 个模板：

1. `低估值`
2. `高股息`
3. `质量成长`

每个模板要有：

- 名称
- 简介
- 默认筛选条件
- 风险说明

---

## 后端要求

后端新增 `v2` 接口，不能污染第一版 `v1` 接口。

必须新增：

### `GET /api/v2/strategies/templates`

返回策略模板列表。

### `GET /api/v2/strategies/templates/{code}`

返回某个模板的默认条件。

### `POST /api/v2/strategies/screen`

根据模板和条件返回股票筛选结果。

---

## 后端实现要求

- 使用当前项目已有 Java/Spring Boot 技术栈
- 新增 controller / service / dto / model
- 与第一版代码隔离清晰
- 先使用 mock 数据或本地静态数据实现 MVP
- 不要一开始接真实数据源

返回结果至少包含：

- 股票代码
- 股票名称
- 行业
- 价格
- PE
- PB
- ROE
- 营收增长率
- 净利润增长率
- 股息率
- 市值
- 入选原因
- 风险标签

---

## 前端要求

沿用现有前端技术栈。

新增第二版页面，但不能破坏第一版页面。

建议：

- 新增独立路由，如 `/strategy` 或 `/v2/strategy`
- 第一版首页可以增加一个进入第二版的入口，但必须是最小改动
- 第一版复利对比页不要被重写

第二版页面至少包含：

1. 页面标题与说明
2. 策略模板卡片或切换区
3. 筛选条件表单
4. 结果表格
5. 风险提示区

---

## 数据要求

第二版 MVP 阶段先不要接真实 A 股接口。

要求：

- 使用本地 mock 数据
- 数据结构尽量贴近未来真实接口
- 让页面和接口完整跑通

mock 数据至少覆盖：

- 10 到 30 只股票
- 3 个策略模板都能筛出结果
- 至少有少量空结果或边界场景可验证

---

## 实施顺序

严格按以下顺序执行：

1. 先阅读现有项目结构
2. 明确第一版边界，不要误改
3. 列出第二版新增文件清单
4. 先完成后端 `v2` 接口
5. 本地验证后端接口
6. 再完成前端第二版页面
7. 接入后端接口
8. 完成联调
9. 补充 README 中的第二版说明

---

## 可接受的改动类型

允许：

- 新增后端 v2 文件
- 新增前端页面与组件
- 新增 mock 数据文件
- 新增路由
- 在首页增加一个进入第二版的按钮
- README 增补说明

不建议：

- 重构第一版目录
- 修改第一版组件结构
- 合并第一版和第二版逻辑
- 提前抽象复杂公共框架

---

## 交付标准

完成后必须满足：

1. 第一版仍然可以正常运行
2. 第一版复利对比功能行为不变
3. 第二版页面可以正常打开
4. 第二版接口可以正常返回数据
5. 第二版页面可以完成一次完整筛选流程
6. 页面显示入选原因和风险提示

---

## 输出要求

开始写代码前，先输出：

1. 第二版新增文件结构
2. 哪些旧文件会被最小修改
3. 为什么这些改动不会影响第一版

完成后，再输出：

1. 新增了哪些文件
2. 修改了哪些旧文件
3. 如何启动前端和后端
4. 如何访问第一版和第二版页面
5. 第一版是否已验证未受影响

---

## 一句话原则

**第二版必须是“增量建设”，不是“覆盖式重做”。**
