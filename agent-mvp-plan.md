# A股复利对比工具 MVP 实施说明

## 1. 目标

构建一个**可运行的最小 MVP**，面向 **A 股小白投资者**，核心能力只有一个：

- 让用户在网页上横向对比多个复利投资方案

本期**不做**：

- 个股选股
- 实时行情
- 回测
- 登录注册
- 推荐买卖
- 复杂权限体系

产出标准：

- 本地可启动
- 前后端可联调
- 用户能在页面输入 2 到 5 个方案并看到对比结果

---

## 2. 用户画像

目标用户：

- 刚开始接触投资的 A 股用户
- 能理解“定投、年化、手续费”，但不懂复杂量化

用户核心问题：

- 每月投多少钱，长期差距有多大
- 不同收益率方案差多少
- 手续费和通胀对长期结果影响多大

---

## 3. MVP 范围

### 3.1 页面

只做 2 个页面：

1. 首页
   - 一句话说明产品用途
   - 一个按钮进入复利对比页

2. 复利对比页
   - 支持新增、删除方案
   - 支持同时对比 2 到 5 个方案
   - 展示结果卡片
   - 展示年度收益曲线图

### 3.2 每个方案的输入项

- 方案名称
- 初始本金
- 每月定投金额
- 投资年限
- 预期年化收益率
- 年费率
- 通胀率

默认值可以给一组新手友好的示例。

### 3.3 输出结果

每个方案至少展示：

- 总投入
- 最终资产
- 总收益
- 收益倍数
- 扣除通胀后的实际购买力

整体比较区至少展示：

- 最优方案
- 年度曲线图
- 2 到 3 条简短结论

---

## 4. 技术约束

## 4.1 后端

- Java 21
- Spring Boot
- Maven
- REST API

建议：

- 先用内存实现，不强制接数据库
- 如果需要持久化，再加 PostgreSQL

本期后端职责：

- 接收多个方案参数
- 执行复利计算
- 返回结构化对比结果

## 4.2 前端

前端框架可由 agent 自主选择，但要求：

- 能快速完成可运行页面
- 表单和图表集成简单

优先建议：

- Next.js 或 Vite + React
- Tailwind CSS
- ECharts 或 Recharts

## 4.3 非功能要求

- 页面在桌面端可正常使用
- 输入校验明确
- 接口报错信息可读
- 代码结构不要过度工程化

## 6. 计算规则

agent 必须统一计算口径，避免前后端不一致。

规则如下：

- 月净收益率 = `(1 + annualReturnRate - annualFeeRate)^(1/12) - 1`
- 按“每月末定投”处理
- 总投入 = 初始本金 + 每月定投金额 * 年限 * 12
- 最终资产按月滚动计算
- 实际购买力 = `finalAmount / (1 + inflationRate)^years`
- 收益倍数 = `finalAmount / totalInvested`

建议至少输出每年的资产节点，用于画图。

## 7. 页面交互要求

## 7.1 首页

展示内容：

- 产品标题
- 一段简短说明
- “开始对比”按钮

## 7.2 复利对比页

页面区块：

1. 顶部说明区
   - 提醒这是测算工具，不构成投资建议

2. 方案输入区
   - 默认显示 2 个方案
   - 可新增到最多 5 个
   - 可删除方案
   - 每个字段都有默认值

3. 计算操作区
   - 一个“开始对比”按钮
   - 一个“恢复默认示例”按钮

4. 结果区
   - 方案结果卡片
   - 最优方案高亮
   - 折线图
   - 简短结论

---

## 8. 默认示例方案

初始化时可直接放 3 组：

1. 银行定存
   - 年化 2%
   - 年费率 0

2. 纯债基金
   - 年化 4%
   - 年费率 0.5%

3. 沪深300 ETF
   - 年化 8%
   - 年费率 0.6%

统一默认：

- 初始本金 50000
- 每月定投 2000
- 投资年限 10
- 通胀率 2.5%

## 10. 实施顺序

agent 应按下面顺序推进，不要跳步骤。

1. 初始化后端项目
2. 实现 `GET /api/health`
3. 实现复利计算 service
4. 实现 `POST /api/v1/compound/compare`
5. 本地自测接口
6. 初始化前端项目
7. 完成首页和复利对比页
8. 接入后端 compare 接口
9. 加入图表和结果卡片
10. 完成基础样式和错误提示
11. 本地联调验证
12. 补充 README 启动说明

---

## 11. 验收标准

满足以下条件才算 MVP 完成：

- 后端可启动
- 前端可启动
- 页面可以输入至少 2 个方案
- 点击按钮后能拿到接口结果
- 页面能展示横向对比结果
- 页面能展示年度曲线图
- 对非法输入有校验
- 项目有明确启动步骤

---

## 12. agent 行为约束

agent 在执行时应遵守：

- 优先保证“能跑起来”，不要先追求完美架构
- 不要引入登录、数据库、缓存、消息队列等非必要能力
- 不要扩展到选股、回测、行情抓取
- 不要加入复杂状态管理，除非前端确实需要
- 不要把页面做成多模块平台
- 不要把收益说明写成投资建议

---

## 13. 完成定义

当以下内容都存在时，可认为任务完成：

- 一个可运行的前端
- 一个可运行的 Java 后端
- 一个可访问的复利对比页面
- 一个可用的 compare API
- 一份说明如何启动前后端的 README

这个 MVP 的唯一目标是：

**让用户真正完成一次复利方案横向对比。**
