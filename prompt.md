# Agent Prompt

你现在要根据以下两份文档生成一个**可运行的最小 MVP**：

- `agent-mvp-plan.md`
- `README_AGENT.md`

## 目标

做一个面向 **A 股小白投资者** 的网页工具，只实现一个核心功能：

**复利方案横向对比**

## 必须完成

1. 一个可启动的前端项目
2. 一个可启动的 Java 后端项目
3. 一个健康检查接口
4. 一个复利对比接口
5. 一个可用页面，支持输入多个方案并展示结果
6. 一个 README，说明如何启动

## 严格范围

只做 MVP，不要扩展到：

- 选股
- 行情
- 回测
- 登录
- 数据库优先设计
- 推荐买卖

## 推荐技术

- 后端：Java 21 + Spring Boot
- 前端：React 方案即可，优先简单可跑
- 图表：ECharts 或 Recharts

## 输出要求

- 代码结构清晰
- 能本地运行
- 前后端能联调
- 输入输出字段一致
- 计算逻辑以后端为准

## 计算口径

- 月净收益率 = `(1 + annualReturnRate - annualFeeRate)^(1/12) - 1`
- 默认每月末定投
- 总投入 = 初始本金 + 每月定投金额 * 年限 * 12
- 实际购买力 = `finalAmount / (1 + inflationRate)^years`

## 执行顺序

1. 先搭后端
2. 再实现计算接口
3. 再搭前端页面
4. 再接接口
5. 最后补 README 和自检

## 完成标准

用户可以输入至少 2 个方案，点击对比后看到：

- 总投入
- 最终资产
- 总收益
- 实际购买力
- 年度曲线图

如果需要做取舍，优先保证“能跑、能算、能看结果”。
